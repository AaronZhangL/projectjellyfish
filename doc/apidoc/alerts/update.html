PUT /alerts/:id
Updates alert with given :id

Endpoint to update an existing alert.
To preserve referential integrity, only the following attributes can be changed: message, start_date, end_date.

Errors

404 Not found.

Examples

PUT /alerts/9
project_id=1&staff_id=0&order_id=2&status=OK&message=Updated
204

Params

Param name Description
staff_id
optional

The staff id of the user who is posting this new alert.
0 indicates a system generated alert.


Value: Must be String
message
optional

The message content to update alert with.


Value: Must be String
start_date
optional

Date this alert will begin appearing. Null indicates the alert will start appearing immediately.


Value: Must be String
end_date
optional

Date this alert should no longer be displayed after. Null indicates the alert does not expire.


Value: Must be String
