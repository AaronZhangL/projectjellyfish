Alerts / create

POST /alerts
Creates a new alert

Endpoint to create a new status alert for an order.
If a status alert is the same as the last one recorded for the service, then the existing status alert is updated.
Otherwise, a new status alert is created for the service.

Examples

POST /alerts
project_id=1&staff_id=2&order_id=3&status=OK&message=NO+ISSUES.
201
{
  "id": 2,
  "project_id": 1,
  "staff_id": 2,
  "order_id": 3,
  "status": "OK",
  "message": "NO ISSUES.",
  "start_date": null,
  "end_date": null,
  "created_at": "2015-01-08T13:35:58.820Z",
  "updated_at": "2015-01-08T13:35:58.820Z"
}

Errors

422 Missing parameter 
--- 
...

Params

  • project_id : required [ Must be String ]

    The project id issued with this new alert.
    A 0 indicates system wide alert.

  • staff_id : required [ Must be String ]

    The staff id of the user who is posting this new alert.
    0 indicates a system generated alert.

  • order_id : required [ Must be String ]

    The order id associated with this new alert.
    A 0 indicates a project wide alert.

  • status : required [ Must be String ]

    HTTP status code issued with this alert.
    Valid Options: OK, WARNING, CRITICAL, UNKNOWN, PENDING

  • message : required [ Must be String ]

    The message content of the new alert.

  • start_date : optional [ Must be String ]

    Date this alert will begin appearing. Null indicates the alert will start appearing immediately.

  • end_date : optional [ Must be String ]

    Date this alert should no longer be displayed after. Null indicates the alert does not expire.
